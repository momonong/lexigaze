# 取得你的專屬 API Key (零成本實戰指南)

在這堂 Vibe Coding 實戰課中，我們將使用 **Google AI Studio** 提供的 API 作為專案的「大腦」，讓它幫我們閱讀學術論文，並轉換成開發指令。

好消息是：這是完全免費的，而且**不需要綁定信用卡**！

## 申請步驟 (只要 1 分鐘)

1. **登入平台**：前往 [Google AI Studio](https://aistudio.google.com/)，使用你的個人 Google 帳號登入。
2. **取得金鑰**：點擊左側導覽列的 **「Get API key」**，接著點擊畫面中間的藍色按鈕 **「Create API key」**。
3. **建立與複製**：點選建立在新專案下（系統會自動命名）。生成後，將那串長長的亂碼金鑰完整複製起來。

---

## 環境變數設定

在真實的軟體開發中，金鑰絕對不能寫死在程式碼裡。請在專案的**根目錄**下，新增一個名為 `.env` 的純文字檔，並將剛剛複製的 Key 貼進去：

```env
GEMINI_API_KEY="AIzaSy...請把你的金鑰貼在這裡..."
```

*(注意：專案的* `.gitignore` *已經設定好會忽略* `.env` *檔，請確保你不要將這個金鑰 commit 到 GitHub 上！)*

---

## 為什麼我們不能共用一把 Key？(Rate Limit 說明)

你可能會想：「既然免費，為什麼老師不直接發一把 Key 給全班用就好？」這牽涉到後端與 API 開發中非常重要的一個機制：**頻率限制 (Rate Limit)**。

> **Google AI Studio 免費版的限制是：每分鐘 15 個請求 (15 RPM)。**

如果我們全班 15 個人共用一把 Key，當大家同時按下 `Enter` 執行解析腳本時，瞬間會產生數十個請求。伺服器為了保護資源，會直接踢回一個 `429 Too Many Requests` 的錯誤代碼，導致全班的程式瞬間當機。

這也是為什麼每個人都需要申請獨立的 Key。

**防爆：** 即使你自己有一把 Key，如果你在 `questions.yaml` 裡一次寫了 20 個問題，程式同時發送出去依然會把你自己的額度打爆。 因此，如果你去偷看 `skill_builder.py` 的原始碼，會發現我們在裡面加了一行 `asyncio.Semaphore(2)` 與睡眠延遲。這是一個「信號量」鎖，用來確保你的電腦同一時間最多只會送出 2 個請求，避開了 15 RPM 的限制
