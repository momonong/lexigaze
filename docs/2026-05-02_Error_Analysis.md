## 3. Root Cause: The "Half-Line Height" Limit
Our EM-based calibration (Skill 6) relies on an initial Viterbi pass to estimate drift. If the hardware drift is greater than half the distance between lines (~30px in this trial), the uncalibrated Viterbi will naturally "snap" to the wrong line. The EM then calculates a drift vector that reinforces this error rather than correcting it.

## 4. Proposed Mitigation Strategies

### A. Non-Linear EM Initialization
Instead of a single-pass EM, we should implement a **Multi-Hypothesis EM**. Initialize 3 drift candidates: `(0, 0)`, `(0, -LineHeight)`, and `(0, +LineHeight)`. Run the window decode for each and pick the one with the highest global Viterbi likelihood.

### B. Anchor-Based Calibration
Leverage "High-Mass" words (Surprisal > 10) as high-confidence anchors. If the gaze jumps suddenly when a high-mass word is expected, it provides a strong signal for drift correction.

### C. Reading Rhythm Prior
Incorporate a "Reading Speed" constraint. Human readers rarely jump a full line and continue reading perfectly at the same pace. Detecting a parallel path on a different line should trigger a "Drift Reset" event.

## 5. Conclusion
While the "Ultimate" pipeline achieved 50% accuracy overall, this analysis shows that the success is likely due to the latter half of the trial where line spacing or text structure becomes less ambiguous. For a truly robust NeurIPS-grade system, we must address the **Line-Locking** failure mode through multi-hypothesis initialization.
