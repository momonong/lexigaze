## 1. The "Fake Results" Bug Identified
During the final verification phase, it was discovered that the evaluation pipeline suffered from a critical "State Space Collapsing" flaw. The system was generating candidate `word_boxes` derived exclusively from the sequence of *fixated* words rather than the complete text layout. This effectively removed unfixated (skipped) words from the decoder's search space, forcing an artificial diagonal bias and guaranteeing near 100% success.

## 2. Architectural Refactoring Completed
To establish a scientifically valid testing environment, the following components were refactored:

### A. Data Extraction Update (`batch_extract_features.py`)
- **Full Layout Extraction**: The script now generates a `layout.csv` for each trial, encompassing every word in the sentence. For words without physical fixations, bounding box coordinates are linearly interpolated.
- **Fixation Mapping**: A separate `fixations.csv` is generated to map actual gaze coordinates (`noisy_x`, `noisy_y`) to their true target index within the full `layout.csv`.

### B. Pipeline Decoupling (`run_full_neurips_pipeline.py` & `evaluate_neurips_final.py`)
- The Viterbi state space (transition matrices, emission boxes, cognitive mass priors) is now instantiated using the **full layout size** (e.g., all 200 words in a paragraph).
- The gaze observation sequence is now fed into the decoder, and the predicted indices are evaluated against the true `layout_index` from `fixations.csv`.
- The model is now properly subjected to the mathematical challenge of deciding when to skip or regress across the *entire* textual environment.
